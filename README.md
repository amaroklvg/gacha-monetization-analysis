# Gacha Monetization Analysis

An end-to-end data analytics portfolio project examining monetization,
player retention and spending behavior in a simulated free-to-play gacha game.

## Project objectives

- Import and profile intentionally dirty data
- Clean and validate the dataset using MySQL
- Analyze player activity, retention and monetization
- Evaluate banner and character performance
- Estimate character profitability and marketing ROI
- Build an interactive Power BI dashboard
- Present business insights and recommendations

## Tools

## Planned analysis

### Monetization

- Average Revenue per User (ARPU)
- Average Revenue per Paying User (ARPPU)
- Payer Conversion Rate
- Observed Lifetime Value
- Time to First Purchase
- Repeat Purchase Rate
- Revenue Concentration

### Engagement and retention

- Daily Active Users
- Monthly Active Users
- DAU/MAU ratio
- D1, D7 and D30 retention

### Gacha behavior

- Banner performance
- Character popularity
- Purchase behavior around banner phases
- Player behavior after losing a 50/50
- Pity-related purchase conversion
- Premium currency sources and sinks

### Character profitability

- Attributed banner revenue
- Character production cost
- Character marketing cost
- Estimated profit
- Return on Investment

## Project workflow

1. Prepare and document the source dataset
2. Load raw data into MySQL staging tables
3. Profile data quality issues
4. Clean and validate the data
5. Build analysis-ready tables
6. Calculate business metrics with SQL
7. Validate selected results in Excel
8. Build a Power BI dashboard
9. Document insights and recommendations

## Dataset

The project uses a synthetic dataset created specifically for this case study.

The exported data contains controlled quality issues, including missing values,
duplicates, inconsistent categories, malformed numbers, orphaned references and
date anomalies.

Hidden behavioral simulation parameters and ready-made player segments are not
included in the analytical dataset.
